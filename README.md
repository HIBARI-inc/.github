# Gitルール

## 1. ブランチ運用
- `main` ブランチは常にデプロイ可能な状態を心がける
- ブランチ名は `<制作物>_<名前>` が基本、必要に応じてナンバリングする

## 2. コミットメッセージ
基本的に以下の３つ＋具体的な変更点の記載
- [add]：機能、データなどの追加
- [fix]：プログラムの修正
- [docs]：資料、補足、ドキュメントの記載

## 3. Pull Request
- フォーマットに従う
- 自分以外のレビュアーを設定してレビューを仰ぐ

## 4. レビュー
- プルリクとREADMEを確認し、確認を行う
- フィードバックをリクエスト送信者に送り、必要に応じてマージする
